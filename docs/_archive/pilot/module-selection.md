## 选择结论
**试点模块**：M02 database（SQLite连接管理）  
**选择理由**：基础设施层、低耦合、高复用、验证核心工程规范（类型注解/错误处理/测试覆盖）  
**风险等级**：🟢 低风险（总分8/20）

| 属性 | 值 |
|------|-----|
| 模块ID | M02 |
| 模块名 | database |
| 功能描述 | SQLite连接生命周期管理（连接池、事务、健康检查） |
| 预估代码行 | 150-200 |

| 与业务耦合 | 无（纯基础设施） |

## 边界冻结（来自module-boundary.md）
**输入**：
- 数据库文件路径（`Path`）
- 配置参数（WAL模式开关、超时时间、最大连接数）

**输出**：
- 连接对象（`sqlite3.Connection`包装）
- 查询结果（`Iterator[sqlite3.Row]`）
- 事务上下文（上下文管理器协议）

**明确包含（Must Have）**：
1. SQLite连接初始化（支持WAL模式）
2. 基础查询接口（execute/query）
3. 连接池管理（单例模式，最大5连接）
4. 事务上下文管理器（with语句支持）
5. 连接健康检查（is_alive）

**明确排除（Must Not Have）** - 阻断条件：
- ❌ 不定义业务表结构（models模块职责）
- ❌ 不处理数据库迁移（migration模块职责）
- ❌ 不拼接业务SQL（防注入在Repository层）
- ❌ 不支持异步IO（第一版仅同步）

## 验收标准（可量化）
| 检查项 | 标准 | 验证方式 |
|--------|------|----------|
| 连接创建 | 能成功连接SQLite文件（不存在时自动创建） | 单元测试 |
| 并发支持 | 10线程并发查询无死锁 | 压力测试 |
| 事务回滚 | 异常时数据不脏 | 单元测试 |
| 连接池 | 耗尽时优雅等待（非崩溃） | 边界测试 |
| 异常处理 | 自定义异常类（非裸抛sqlite3.Error） | 代码审查 |
| 代码质量 | 函数&lt;50行，圈复杂度&lt;10，类型注解100% | mypy+ruff |
| 测试覆盖 | &gt;80% | pytest --cov |
